What happens when you initialize a repository? Why do you need to do it?

By using the command "git init" you are basically telling Git to create a repository within the current directory. A ".git" file is created within the current directory which allows commits to now be executed.

How is the staging area different from the working directory and the repository?
What value do you think it offers?

The staging area serves as intermediary step between the working directory and the repository. The benefits of a staging area is that it allows a more granular control of which specific files to include when making a commit to the repository.
